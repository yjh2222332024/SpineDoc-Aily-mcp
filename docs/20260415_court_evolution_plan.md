# 📜 SpineDoc 联邦法庭与星系治理规划 (2026-04-15)

## 1. 现状断言

当前星系 (Galaxy) 的生成采用"锚点硬匹配"策略。虽然这导致了星系物理碎片化，但在当前文档量级下，它提供了优秀的物理可解释性与低耦合度。我们决定拒绝对写入流进行过度优化，以避免人为引入复杂性与数据漂移风险。

## 2. 核心决策：知识代谢的"异步原则"

为了解决未来星系数量爆炸与查询召回瓶颈，我们确立以下"解耦"纪律：

### 写入流确定性 (Deterministic Ingest)
- 保持现有 Ingest 逻辑，不对 Galaxy 表进行动态合并
- 保证数据录入的原子性与速度

### 异步知识代谢 (Asynchronous Metabolism)
- 将星系合并逻辑剥离至后台脚本 `scripts/realign_real_galaxies.py`
- 该脚本定期（例如每日）扫描 Galaxy 质心矩阵，识别相似度 > 0.95 的星系
- **结果呈现**：生成"Thesaurus Map"（同义星系映射表），而非直接物理篡改星系结构

### 智能召回 (Intelligent Retrieval)
- 联邦法庭 (Federated Court) 的 Distributor 节点在执行任务时，必须优先查阅 Thesaurus Map
- 将逻辑相近的"碎片星系"视为同一个召回源，在逻辑层实现统一
