## 文档管理

### 导入文档

```bash
spine ingest <pdf 文件> [选项]
```

**参数：**
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `path` | PDF 文件或目录路径 | 必需 |
| `-f, --force` | 强制重新入库 | `False` |
| `-l, --limit` | 限制处理页数 | 无限制 |
| `-t, --toc-range` | 手动指定目录范围 | 自动检测 |
| `-s, --stop-words` | 自定义停用词文件 | 内置词表 |
| `-k, --key` | 注入私有 API Key | 使用.env 配置 |
| `-o, --ocr` | 强制使用 OCR 模式 | 自动检测 |
| `-F, --font-feature` | 使用字体特征提取 | `False` |
| `--dev` | 开发者模式（强制删除 Checkpoint） | `False` |

**示例：**
```bash
# 导入单个 PDF
spine ingest contract.pdf

# 导入目录中的所有 PDF
spine ingest ./documents/

# 强制重新导入（覆盖已有数据）
spine ingest contract.pdf --force

# 只处理前 10 页
spine ingest paper.pdf --limit 10

# 强制使用 OCR（适合扫描件）
spine ingest scanned_doc.pdf --ocr

# 使用自定义停用词
spine ingest paper.pdf --stop-words ./medical_terms.txt

# 只处理特定目录范围（如第 9-15 页）
spine ingest paper.pdf --toc-range 9,15
```

**输出示例：**

## 问答查询

### 提问

```bash
spine ask "<问题>" [选项]
```

**参数：**
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `query` | 你的问题 | 必需 |
| `-d, --doc` | 目标文档 ID | `all` (所有文档) |
| `-l, --limit` | 召回上限 | `15` |
| `-k, --key` | 注入私有 API Key | 使用.env 配置 |
| `-o, --online` | 激活联网证人 | `False` |

**示例：**
```bash
# 多文档检索
spine ask "SM4 和 AES 的主要区别是什么？"

# 单文档检索
spine ask "SM4 的密钥长度" -d 9b1d1195

# 联网搜索（可能更新知识库）
spine ask "最新的密码学标准" --online

# 限制召回数量
spine ask "分组密码工作模式" --limit 5

# 组合使用
spine ask "SM4 的性能指标" -d 9b1d1195 --limit 10 --online
```

### 查看语义切片

```bash
spine chunks <文档 ID> [选项]
```

**参数：**
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `doc_id` | 文档 ID | 必需 |
| `-l, --limit` | 显示 Chunk 数量 | `20` |
| `-p, --page` | 筛选特定页码 | 无筛选 |
| `-k, --keyword` | 筛选关键词 | 无筛选 |

**示例：**
```bash
# 查看前 20 个切片
spine chunks 9b1d1195

# 查看前 5 个切片
spine chunks 9b1d1195 --limit 5

# 只看第 10 页的切片
spine chunks 9b1d1195 --page 10

# 筛选包含"加密"的切片
spine chunks 9b1d1195 --keyword 加密

# 多关键词筛选
spine chunks 9b1d1195 --keyword "加密，解密"
```

## Git 版本控制

### 查看 Git 历史

```bash
spine git history <ChunkID> [选项]
```

**参数：**
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `chunk_id` | Chunk ID | 必需 |
| `-l, --limit` | 显示历史记录数量 | `20` |

**示例：**
```bash
spine git history 6021e448
spine git history 6021e448 --limit 10
```

**输出示例：**
```
╭──────────────────────────────────────────────────────────────╮
│ 🔍 查询 Chunk: 6021e448...                                   │
╰──────────────────────────────────────────────────────────────╯

┌───────────────────── 📜 Chunk 版本历史 ────────────────────────┐

│ a3f5d2c1  │ 2026-04-16T…   │ yjh66…  │ 更新 SM4 密钥长度描述  │

└───────────┴────────────────┴─────────┴───────────────────────┘
```

### 查看指定版本

```bash
spine git show <ChunkID> [选项]
```

**参数：**
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `chunk_id` | Chunk ID | 必需 |
| `-t, --to` | 查看指定提交版本 | 当前版本 |

**示例：**
```bash
# 查看当前版本
spine git show 6021e448

# 查看历史版本
spine git show 6021e448 --to a3f5d2c1
```

### 回滚到指定版本

```bash
spine git revert <ChunkID> --to <commit_hash> [选项]
```

**参数：**
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `chunk_id` | Chunk ID | 必需 |
| `--to` | 目标提交哈希 | 必需 |
| `-y, --yes` | 确认回滚（跳过确认） | `False` |

**示例：**
```bash
# 回滚（带确认提示）
spine git revert 6021e448 --to a3f5d2c1

# 回滚（跳过确认）
spine git revert 6021e448 --to a3f5d2c1 --yes
```

### 比较差异

```bash
spine git diff <ChunkID> -o <old_commit> -n <new_commit>
```

**参数：**
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `chunk_id` | Chunk ID | 必需 |
| `-o, --old` | 旧提交哈希 | 必需 |
| `-n, --new` | 新提交哈希 | 必需 |

**示例：**
```bash
spine git diff 6021e448 -o b8e4f1a9 -n a3f5d2c1
```

**输出示例：**

```bash

```



## 最佳实践

### 1. 导入文档

```bash
# 批量导入
spine ingest ./documents/

# 使用 OCR 处理扫描件
spine ingest scanned_contract.pdf --ocr

# 限制处理页数（快速测试）
spine ingest large_paper.pdf --limit 20
```

### 2. 问答查询

```bash
# 精确查询单文档
spine ask "核心论点" -d 9b1d1195

# 跨文档对比
spine ask "对比 SM4 和 AES 的安全性"

# 启用联网搜索获取最新信息
spine ask "2026 年密码学标准更新" --online
```

### 3. 审计文档

```bash
# 1. 导入文档
spine ingest contract.pdf

# 2. 查看脊梁确认结构
spine tree <文档 ID>

# 3. 查询关键条款
spine ask "违约责任条款" -d <文档 ID>

# 4. 检查潜在矛盾
spine ask "是否存在自相矛盾的条款" -d <文档 ID> --online

# 5. 查看 Git 历史（如有修改）
spine git history <ChunkID>
```

### 4. 性能优化

```bash
# 限制召回数量提高响应速度
spine ask "问题" --limit 5

# 单文档查询避免不必要的全局检索
spine ask "问题" -d <文档 ID>

# 使用关键词筛选快速定位
spine chunks <文档 ID> --keyword "关键术语"
```

## 快捷键和技巧

### 终端技巧

| 技巧 | 说明 |
|------|------|
| `spine --help` | 显示完整帮助 |
| `spine <command> --help` | 显示命令帮助 |
| `Tab` 键 | 自动补全命令和参数 |
| `Ctrl+C` | 中断长时间运行的命令 |

### 文档 ID 快捷方式

```bash
# 使用前缀即可（无需完整 ID）
spine tree 9b1d  # 等同于 9b1d1195-51c5-44ee-a265-8f29edb199cb

# list 命令输出中的 ID 前 8 位可直接用于其他命令
spine list  # 查看 ID
spine tree 9b1d1195  # 使用看到的 ID
```
