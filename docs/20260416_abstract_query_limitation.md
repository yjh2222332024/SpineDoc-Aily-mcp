# 抽象问题回答能力局限分析

**日期**: 2026-04-16  
**问题**: 系统对"0 基础入门 tamarin"等抽象问题的回答能力非常有限

---

## 问题现象

用户提问：`0 基础入门 tamarin`

**系统返回**:
```
根据提供的【Source】片段，本文件未提供关于"0 基础入门 tamarin"的完整入门指南。
文件仅包含以下相关信息：
1. 从源代码编译安装 (P11)
2. 在命令行运行 Tamarin (P30)
3. Tamarin-Troop 工具 (P139)
文件没有提供关于如何编写第一个理论文件、基本概念解释或系统性的学习路径等零基础入门所需的核心信息。
```

**用户评价**: "这个系统对于抽象问题的回答非常有限"

---

## 根本原因分析

### 1. LLM 温度过低

当前全链路温度配置：
| 节点 | 温度 | 职责 |
|------|------|------|
| Scout | 0.2 | 查询拆解 |
| Examiner | 0.1 | 证据选择 |
| Integrator (单文档) | 0.1 | 答案合成 ← 问题所在 |
| Moderator (联邦) | 0.1 | 冲突裁决 + 答案合成 ← 问题所在 |

**问题**: 答案合成阶段温度 0.1，导致 LLM 变成机械的"证据复读机"，无法：
- 把分散的安装、运行、示例步骤串联成连贯的"入门路径"
- 进行创造性的信息整合
- 生成有结构、有层次的指导性内容

### 2. 架构设计问题

当前 `Moderator` 一身兼二职：
1. **冲突裁决**：需要低温严谨（0.1 合理）
2. **答案整合**：需要创造性整合（0.1 太死板）

这两个职责需要不同的温度策略，但目前混在同一个节点。

### 3. 检索不是问题

实际检索到了正确的内容：
- P11: 安装说明
- P30: 命令行运行
- P139: Tamarin-Troop 工具

**问题不在检索，在整合**。

---

## 解决方案

### 方案 A: 新增 Synthesizer 节点（推荐）

```
联邦法庭流程:
1. Distributor → 传唤证人
2. Collector → 收集证据
3. Moderator → 裁决冲突 (温度 0.1)
4. Synthesizer → 整合答案 (温度 0.6) ← 新增
```

**Synthesizer 职责**:
- 输入：证据包 + 冲突裁决结果
- 输出：结构化、可读性强的最终答案
- Prompt: "你是一个外交官/记录官，把法庭证据整合成一份通俗易懂的指南"
- 温度：0.6-0.7

### 方案 B: 单文档 Integrator 提温

将 `nodes.py:integrator_node` 的温度从 0.1 提升到 0.6，允许 LLM 进行基本的信息串联。

### 方案 C: 双管齐下

同时实施方案 A 和 B。
