# 🏛️ SpineDoc V1.0.0 "联邦刺客" 架构深度解析

## 1. 核心分层模型

SpineDoc 采用 **“三级金字塔”** 检索与推理模型，将海量非结构化数据蒸馏为硬核事实。

### L1：星系层 (Galaxy Layer) - 机械粗筛
- **组件**: Galaxy Radar (星系雷达)
- **原理**: 基于 **TF-IDF 关键词位掩码**。
- **作用**: 在毫秒级时间内，从万级文档库中瞬间定位到 3-5 份相关星系，剔除 90% 以上的干扰文档。

### L2：星云层 (Nebula Layer) - 航道锁定
- **组件**: Cascading Retriever (级联检索器)
- **原理**: 基于 **TOC 标题 + 章节关键词** 的双路向量搜索。
- **作用**: 精准锁定文档内的物理页码区间 [P_start, P_end]，确保证据不在目录或不相关章节中迷失。

### L3：尘埃层 (Dust Layer) - 联邦辩论
- **组件**: Federated Court (联邦法庭)
- **原理**: 基于 **LangGraph 编排的多智能体博弈**。
- **作用**: 通过蒙眼取证、冲突嗅探和回溯质询，压榨出绝对客观的原子事实。

## 2. 关键自研协议

### ⚓ 铁锚协议 (Iron Anchor Protocol)
- **问题**: PDF 逻辑页码（书上印的）与物理页码（渲染出的）通常存在 40+ 页的巨大偏移。
- **解法**: 自动搜索所有 TOC 项中的“Page 1”候选点，利用**中值原点标定技术**（Median Calibration）实现物理坐标的像素级对齐。

### 🥷 逻辑刺客协议 (Logic Assassin Protocol)
- **核心**: **负向约束指令集**。
- **逻辑**: 严禁 Agent 进行逻辑补全、严禁情感介入、严禁无锚点引用。如果同一逻辑点辩论 3 轮未果，强制熔断并提交争议报告。

## 3. 技术栈
- **推理编排**: LangGraph
- **底座服务**: FastAPI, SQLModel
- **向量存储**: PostgreSQL + pgvector
- **视觉感知**: BodyAlchemist (OCR) + PyMuPDF
